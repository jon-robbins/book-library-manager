🔍 Filter Behavior Specifications

Author Filter Logic

- Multi-select checkboxes
- Dynamically extracted from books
- Alphabetical A-Z sorting
- AND operation: book.author IN selectedAuthors
- Empty authors: "No authors available" message

Year Range Filter Logic

- Min/Max number inputs (YYYY format)
- Validation: yearMin ≥ 1900
- Validation: yearMax ≤ latest year in data
- Validation: yearMin ≤ yearMax
- If min only: book.year ≥ min
- If max only: book.year ≤ max
- If both: book.year ≥ min AND book.year ≤ max
- Invalid dates: included in results

Combined Filter Pipeline

result = books
  .filter(search_query_match)    // Text search
  .filter(author_match)           // If authors selected
  .filter(year_range_match)       // If year range set
  .sort(by_sort_order)            // Apply sorting

🚀 Integration Points

With Existing Code

  • TilingDashboard - Parent component, manages filter visibility and state
  • BookTile - No changes needed, displays filtered books
  • CSS Grid - Responsive layout unchanged, controls adjusted
  • Firebase DataConnect - Uses existing useListMyBooks() hook

Event Flow

1. User clicks "Filters" button
2. handleToggleAdvancedFilters updates showAdvancedFilters
3. Filter panel renders with current values
4. User selects authors/years
5. On "Apply": handleAdvancedFiltersChange updates advancedFilters
6. useEffect updates advancedFilters dependency
7. Filtering pipeline recalculates
8. filteredBooks updates with results
9. UI re-renders with filtered books
